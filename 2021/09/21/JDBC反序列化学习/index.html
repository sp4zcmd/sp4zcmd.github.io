漏洞分析

分析ServerStatusDiffInterceptor

在调用com.mysql.cj.jdbc中相关函数来连接恶意MySQL服务器时，会调用到com.mysql.cj.jdbc.result.ResultSetImpl.getObject()函数，而其中会调用readObject()函数执行反序列化操作

先看反序列化点

com.mysql.cj.jdbc.result.ResultSetImpl.getObject()

  1. 判断输入参数columnIndex是否大于1，如果是就向下执行
  2. 判断是不是二进制数据（isBinary），还有Mysql字段类型是否为BLOB（isBlob）
  3. 调用getBytes从Mysql服务端获取对应的字节码数据
  4. 判断输入JDBC链接URL的autoDeserialize是否为true，满足则进入反序列化操作
  5. 判断前两个字节是否为-84和-19，即AC和ED
  6. 满足条件后进入反序列化操作

getBytes读取Mysql恶意服务端上的字节码，用于反序列化

com.mysql.cj.jdbc.result.ResultSetImpl.getObject()是怎么被调用的？

  1. 在com.mysql.jdbc.Driver中新建连接实例
  2. 设置对应的查询拦截器，也就是在JDBC连接url中设置的ServerStatusDiffInterceptor
  3. 从MySQL服务端初始化Properties并执行相关的sql语句，如果查询拦截器不为空则调用查询拦截器的preProcess函数
  4. 运行查询语句SHOW SESSION STATUS，然后调用ResultSetUtil.resultSetToMap()，其中调用了getObject()函数触发反序列化（注意columnIndex为2处才能走到反序列化的代码逻辑，因为为1则直接返回null）
  5. 进入getObject()，判断字段类型为BLOB

1.

java.sql.DriverManager#getConnection

新建连接实例

2.

com.mysql.cj.jdbc.ConnectionImpl#connectOneTryOnly

调用setQueryIntercptors，设置拦截器（ServerStatusDiffInterceptor）

com.mysql.cj.jdbc.ConnectionImpl#setAutoCommit

这里执行了一些诸如set autocommit=1之类的sql语句，进一步触发之前设置好的拦截器

3.

com.mysql.cj.protocol.a.NativeProtocol#sendQueryPacket

这里调用了拦截器（ServerStatusDiffContext）的preProcess方法

4.

com.mysql.cj.jdbc.interceptors.ServerStatusDiffInterceptor#preProcess

调用populateMapWithSessionStatusValues

5.

com.mysql.cj.jdbc.interceptors.ServerStatusDiffIntercept#populateMapWithSessionStatusValues

这里执行了sql语句SHOW SESSION STATUS

然后调用ResultSetUtil.resultSetToMap()

6.

com.mysql.cj.jdbc.util.ResultSetUtil.resultSetToMap

调用com.mysql.cj.jdbc.result.ResultSetImpl#getObject()，触发反序列化

调用栈

总结

利用前提：JDBC链接可控

JDBC初始化过程中会执行某些sql语句（SHOW SESSION STATUS），进而触发拦截器（ServerStatusDiffInterceptor）的preProcess方法，最后调用getObject触发反序列化

  1. com.mysql.cj.jdbc.interceptors.ServerStatusDiffInterceptor#preProcess
  2. com.mysql.cj.jdbc.interceptors.ServerStatusDiffIntercept#populateMapWithSessionStatusValues
  3. com.mysql.cj.jdbc.util.ResultSetUtil#resultSetToMap
  4. com.mysql.cj.jdbc.result.ResultSetImpl#getObject()

终点：在com.mysql.cj.jdbc.result.ResultSetImpl#getObject()处读取Mysql恶意服务端上的设置好的序列化字节码，然后执行readObject触发反序列化
