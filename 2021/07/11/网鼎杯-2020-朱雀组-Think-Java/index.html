dbName直接被拼接到了sql语句中，存在sql注入，在源码中可以看到数据库名为myapp

但是需要先保证数据库链接语句不出错，数据库名必须正确，后面的语句可以用?绕过或者#绕过

#绕过

HTTP请求不包括#，#是用来指导浏览器动作的，对服务器端完全无用。所以，HTTP请求中不包括#。
比如，访问下面的网址，http://www.example.com/index.html#print，浏览器实际发出的请求是这样的：

GET /index.html HTTP/1.1

Host: www.example.com

那么我们向服务器端请求还是#号之前的内容，#后面的内容用来指导浏览器动作。

?绕过

jdbc连接数据库语句后面可以跟参数
jdbc:mysql://localhost:3306/数据库名?user=用户名&password=密码&useUnicode=true&characterEncoding=utf-8&serverTimezone=GMT
后跟无效参数也不会影响
